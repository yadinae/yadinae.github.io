GitHub Pages 博客部署指南

博客类型: 技术分享 | 代码版本: 基于 Hexo + GitHub Pages

1. Introduction (problem + scope)

为什么需要 GitHub Pages？在 2026 年，构建个人技术博客需要考虑以下痛点：

  • ❌ 传统服务器部署成本高（域名、VPS、维护）
  • ❌ 静态网站托管需要第三方服务（Netlify、Vercel）
  • ❌ 难以控制完全的定制化

GitHub Pages 的优势：

  • ✅ 完全免费（个人使用）
  • ✅ 零配置（自动部署）
  • ✅ 强大的 Git 工作流
  • ✅ 集成 GitHub Actions
  • ✅ 支持自定义域名和 HTTPS

本文将手把手教你从零搭建一个基于 Hexo + GitHub Pages 的技术博客，涵盖：

  • 环境准备
  • Hexo 安装与配置
  • 主题定制
  • 部署到 GitHub Pages
  • SEO 优化
  • 高级功能（自定义域名、HTTPS）

3.1 整体架构图

数据流向：

  1. 在本地编辑 Markdown 文章
  2. 运行 hexo generate 生成 HTML/CSS/JS
  3. Git 提交更改到 GitHub 仓库
  4. GitHub Pages 监听仓库变化，自动重新部署
  5. 完成后访问 https://username.github.io

3.2 步骤 1：创建 GitHub 仓库

仓库命名规则：

  • 必须是 username.github.io 格式（username 替换为你的 GitHub 用户名）
  • 如果是 username.github.io 仓库，访问地址是 https://username.github.io
  • 其他仓库访问地址是 https://username.github.io/repo-name

3.4 步骤 3：配置 GitHub Pages

方式 1：GitHub 网页配置（推荐新手）

  1. 访问你的 GitHub 仓库
  2. 点击 Settings → Pages
  3. 在 Build and deployment → Source 中选择：

  4. 点击 Save
  5. 等待 1-2 分钟，状态变为 Published
  6. 访问 https://username.github.io

8. 对比分析

8.1 GitHub Pages vs Vercel

特性 GitHub Pages Vercel
免费额度 无限（个人） 100GB 带宽/月
自动部署 ✅（通过 Git） ✅（通过 Git）
自定义域名 ✅ ✅
HTTPS ✅ ✅
CDN ✅（自动） ✅（自动）
适合场景 静态博客 全栈应用

推荐：

  • 纯静态博客 → GitHub Pages（免费、稳定）
  • 全栈应用 → Vercel（功能更强大）

8.2 静态博客 vs 传统 CMS

特性 静态博客 传统 CMS
部署难度 低（一键部署） 高（服务器配置）
性能 高（纯静态） 中（动态渲染）
安全性 高（无数据库） 中（需防护）
维护成本 低 高
扩展性 中（插件有限） 高（无限）
SEO 优秀 优秀

推荐：

  • 个人博客、技术文档 → 静态博客
  • 电商、内容平台 → 传统 CMS
