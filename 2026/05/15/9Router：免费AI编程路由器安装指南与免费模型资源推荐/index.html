发表于2026-05-15|更新于2026-05-15|AI工具

什么是 9Router？

9Router 是一个开源的 AI 路由器，它可以作为代理层连接你的 AI 编程工具（Claude Code、Codex、Cursor、Cline、Copilot 等）到 40+ 个 AI 提供商和 100+ 个模型。核心卖点：

  • RTK Token Saver：自动压缩工具输出（git diff、grep、ls 等），节省 20-40% 输入 Tokens
  • 三级自动回退：订阅 → 低价 → 免费，零停机
  • 格式翻译：OpenAI ↔ Claude ↔ Gemini 格式自动转换
  • 配额追踪：实时显示各提供商剩余额度
  • 多账户轮询：同一提供商多个账户自动负载均衡

重点：9Router 安装到哪儿最合适？

这是很多新手最迷惑的问题。根据使用场景，建议如下：

场景一：个人开发机（推荐新手）

建议：本机直接安装（npm global）

  • 优点：零配置、延迟最低、调试方便
  • 缺点：关机关服

适合个人日常编码使用，http://localhost:20128 只有本机能访问。

场景二：局域网服务器/NAS

建议：Docker 部署到局域网机器，绑定 0.0.0.0

  • 优点：团队共享、持续运行、集中管理配置
  • 缺点：需要一台常开设备

💡 数据持久化：/app/data 映射到宿主机目录，升级容器不会丢配置。

场景三：云服务器/VPS

建议：Docker + Nginx 反向代理 + 域名

  • 优点：随时随地访问、稳定运行
  • 缺点：需要云服务器成本（推荐低价轻量云，如阿里云 ¥24/月）

场景四：本地开发机 + 配置文件集中管理

9Router 默认数据目录：

  • Linux/macOS：~/.9router/
  • Windows：%APPDATA%/9router/

如需自定义位置，设置环境变量：

推荐：将数据目录放在 SSD 固态硬盘上，因为 9Router 需要频繁读写请求日志和配额数据，HDD 会影响响应速度。

一句话总结

人群 推荐 原因
个人开发者 本机 npm install 简单直接，延迟最低
小团队 局域网 Docker 共享配置，持续运行
重度用户 / 远程办公 云服务器 Docker + Nginx 随时访问，稳定可靠
容器化重度用户 Kubernetes / Docker Swarm 弹性伸缩，高可用

总结

9Router 解决了 AI 编程中一个非常实际的问题：配额管理和 Token 浪费。通过自动回退、Token 压缩和多 Provider 负载均衡，让开发者可以专注于编码本身。

如果你是 AI 辅助编程的重度用户，9Router 值得一试。安装只需一条命令，配置只需几分钟，换来的却是无忧的编码体验。而选择安装位置，核心思路就是一句话：面向自己用就装本地，面向团队用就上服务端。
