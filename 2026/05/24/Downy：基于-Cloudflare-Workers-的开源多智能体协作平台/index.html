Downy：基于 Cloudflare Workers 的开源多智能体协作平台

发表于2026-05-24|更新于2026-05-24|技术教程

如果你一直在寻找一种既能拥有强大的 AI 代理团队，又不想被复杂的服务器运维搞得头大，那么 Downy 绝对值得你关注。作为 OpenClaw 的有力替代者，它巧妙地利用了 Cloudflare Workers 的边缘计算能力，让你在几分钟内就能部署出一支属于自己的 AI 军团。

为什么选择 Downy？

很多 AI 代理平台虽然功能强大，但往往需要繁琐的 Docker 配置或昂贵的云服务器。Downy 走了一条完全不同的路：全托管、零服务器、极致安全。

它把每一个 AI 代理（Agent）都设计成了一个独立的 Durable Object。这意味着每个代理都有自己的“大脑”（对话历史）和“储物柜”（R2 存储空间），互不干扰，且响应极快。

核心亮点

  • 🧠 多代理协作：主代理指挥，子代理执行，模型配置随心切换。
  • ☁️ 边缘部署：跑在 Cloudflare Workers 上，全球加速，无需担心服务器宕机。
  • 🔐 自带护卫：强制集成 Cloudflare Access，只有你能访问你的代理。
  • 🔍 自带搜索：集成 Exa API，让你的代理能实时联网查资料。
  • 🤖 模型自由：无论是 Kimi、OpenRouter 还是 ChatGPT Plus，都能轻松接入。

快速上手指南

部署 Downy 并不复杂，你只需要准备好 Cloudflare 账号和几个 API Key。

  1. 环境准备：确保本地有 Node.js 24 和 pnpm。
  2. 克隆代码：git clone https://github.com/bensenescu/downy
  3. 配置环境：复制 .env.example 并填入你的 TEAM_DOMAIN 和 OPENROUTER_API_KEY。
  4. 一键部署：这里不用传统的 Wrangler，而是使用更智能的 alchemy 工具。只需运行 pnpm deploy，剩下的交给自动化脚本。

值得注意的小细节

  • 免费计划友好：如果你使用 OpenRouter，完全可以绕过 Workers AI 的付费限制。
  • 安全第一：部署后记得开启 Cloudflare Access，否则 Worker 会拒绝所有访问。
  • 独立空间：每个 Agent 的文件都存在 R2 里，你可以像管理本地文件一样管理它们的知识库。

项目地址：github.com/bensenescu/downy
