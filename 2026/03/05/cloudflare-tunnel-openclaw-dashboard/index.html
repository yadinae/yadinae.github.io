通过 Cloudflare Tunnel 安全访问 OpenClaw Dashboard 完整指南

摘要：本文详细介绍如何通过 Cloudflare Tunnel 将 OpenClaw Dashboard 安全地暴露到公网，实现随时随地访问你的 AI 助手。包含完整的配置步骤、认证配对流程以及常见问题排查。

关键词：OpenClaw, Cloudflare Tunnel, Dashboard, 远程访问，安全配置

前言

OpenClaw 是一个强大的 AI 助手框架，默认情况下其 Dashboard 只能在本地访问。但在实际使用中，我们经常需要在外出时通过手机或笔记本访问 Dashboard，与 AI 助手对话。

直接开放端口到公网存在安全风险，而使用 Cloudflare Tunnel 可以在不开放任何端口的情况下，安全地将服务暴露到公网。

本文你将学到：

  • 如何配置 Cloudflare Tunnel
  • 如何完成 Dashboard 认证与设备配对
  • 常见问题的排查方法

前置条件：

  • 已安装 OpenClaw 并可以本地访问
  • 拥有一个域名（已添加到 Cloudflare）
  • 一台可公网访问的服务器（本文使用阿里云 Ubuntu 22.04）

一、为什么选择 Cloudflare Tunnel

传统端口映射的问题

  • ❌ 需要开放防火墙端口
  • ❌ 暴露服务器 IP
  • ❌ 容易受到扫描和攻击
  • ❌ 需要自己处理 HTTPS 证书

Cloudflare Tunnel 的优势

  • ✅ 无需开放任何端口
  • ✅ 隐藏服务器真实 IP
  • ✅ 自动 HTTPS（由 Cloudflare 提供）
  • ✅ 内置 DDoS 防护
  • ✅ 免费使用

二、架构说明

关键安全设计：

  1. OpenClaw Gateway 仅监听 127.0.0.1:10719（不暴露公网）
  2. Cloudflare Tunnel 作为唯一入口
  3. Dashboard 使用 Token 认证
  4. 每个设备需要单独配对批准

五、重要说明

5.1 设备配对机制

  • 每台设备需要单独配对：换一台电脑需要重新执行 4.1-4.3 步骤
  • 每个浏览器需要单独配对：同一台电脑，Chrome → Firefox 也需要重新配对
  • 配对信息持久化：已配对的设备无需重复操作

安全意义：即使他人知道你的 Dashboard URL 和 Token，也无法在未配对的情况下访问你的 Gateway。

七、安全建议

  1. Token 安全

    • Token 相当于密码，不要分享给他人
    • 不要提交到版本控制系统
    • 定期更换 Token

  2. 设备管理

    • 定期检查已配对设备列表
    • 及时撤销不再使用的设备权限
    • 公共电脑上不要配对

  3. Gateway 配置

    • bind 始终设置为 loopback
    • 不要直接暴露 Gateway 到公网
    • 使用强随机 Token

  4. 日志监控

    • 定期检查 Cloudflare Tunnel 日志
    • 关注异常连接尝试

八、完整流程检查清单

部署完成后，按以下清单逐项验证：

  • cloudflared 安装成功
  • Tunnel 创建成功
  • 配置文件正确（Tunnel ID、域名、端口）
  • systemd 服务运行正常
  • 可以通过 https://你的域名 访问 Dashboard
  • 已获取 Token（openclaw dashboard-no-open）
  • 已在 Dashboard 概览页面配置 Token
  • 已批准设备配对（openclaw devices approve）
  • Dashboard 聊天页面可以发送消息
  • 可以收到 AI 回复
