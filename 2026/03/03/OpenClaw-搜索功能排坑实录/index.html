一、毛坯 OpenClaw 并不聪明

网上很多文章都在讲 OpenClaw（大龙虾）有什么能力、技能，有多强。也有很多非 IT 人员跟风在本地，甚至专门买了云主机去部署 OpenClaw，部署很简单，一键脚本就安装完毕，最多再配置个钉钉、飞书之类的通讯工具，你能和 OpenClaw 交流了，你以为你已经有了 24 小时 AI 助理，但实际上，他连基本的 web 搜索能力还没有。

作为非 IT 人员，你给大龙虾一个指令”搜索……新闻；搜索……”，结果什么结果也没有。你问龙虾有没有搜索能力，他自信满满的告诉你他有搜索能力，你给他一个具体搜索任务，他又掉链子。

他告诉你可以配置”Brave Search API”，Brave Search API 确实每天有免费额度，但是：

  1. 国内网络访问不了 Brave
  2. 就算你”想尽办法”可以访问 Brave，想白嫖他的免费额度，还得绑定一个国外银行卡
  3. 你折腾半天又回到原点

当你指出他的”无能”时，他倒是很诚实，也能给你提出第二方案，但他只管给方案，从不管方案可行不可行。

他让我去阿里百炼 MCP 市场再去搞个 MCP 的 key，看到这里，我突然想你你们单位的领导，只管一本正经的拍脑袋，从不管事情能不能干得通。

走到这一步，我就失去耐心了。

二、再给 OpenClaw 装个脑子——Claude Code

由于刚部署的大龙虾并不聪明，于是我自己脑洞大开，给 Ubuntu 服务器安装一个 Claude code，让 Claude 去运维包括 openclaw 在内的所有 AI 应用。

首先说明，我的 OpenClaw 和 Claude 都使用的是阿里 qwen3-max-2026-01-23 模型。

先看看 Claude 对国内网络配置搜索 brave api 的理解，同样模型下，Claude 确实比 OpenClaw 对任务的理解能力更强。

无需梯子的替代方案

既然你在阿里云服务器上运行 OpenClaw，可以考虑以下方案：

方案 1: 使用阿里云百炼的搜索能力
你之前提到的阿里云百炼文档，可能就是关于如何在百炼平台上配置搜索/RAG 能力。百炼在国内可访问，但需要确认它是否支持与 OpenClaw 集成。

方案 2: 使用国内 API

  • 阿里云 Dashscope - 你已经在用（qwen3.5-plus），但它本身不带网络搜索功能
  • 百度智能云 - 有搜索 API
  • 腾讯云 - 有搜索相关服务

方案 3: 服务器配置代理
在你的阿里云服务器上配置代理/梯子，让 API 请求通过代理访问。

方案 4: 使用阿里云百炼应用
你之前给的链接可能就是关于如何在百炼上创建带搜索能力的应用，然后通过 API 调用。

六、经验总结

关键决策点

  1. 网络环境约束：优先选择国内可访问的服务，避免生产环境依赖不稳定网络
  2. 框架限制识别：当内置功能不支持时，及时切换到自定义扩展方案
  3. API 验证先行：在集成到框架前，先用脚本验证 API 可用性

技术要点

项目 说明
API Provider 阿里云百炼 Web-Search MCP
Endpoint https://dashscope.aliyuncs.com/api/v1/mcps/WebSearch/mcp
认证方式 Bearer Token (API Key)
协议 JSON-RPC 2.0
服务名称 bailian_web_search

成本与限制

  • 免费额度：每月 2000 次调用
  • 限流：单用户 5 QPS
  • 适用场景：实时新闻、天气、股票、金价、油价等时效信息

七、后续优化方向

  1. 技能集成：将脚本调用封装为 OpenClaw 原生 Tool，让 AI 模型自动调用
  2. 结果缓存：相同查询结果缓存 15-30 分钟，减少 API 调用
  3. 错误处理：添加重试机制和额度监控告警
