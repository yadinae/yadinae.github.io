摘要

本文总结博客配置过程中遇到的三个问题及其解决方案：标签页空白、分类页空白、Follow Me 链接错误。通过创建自定义布局文件和使用正确的 Pug 语法，成功修复所有问题。

问题概述

问题一：Follow Me 链接错误

现象：博客侧边栏的”Follow Me”按钮链接到 https://github.com/xxxxxx

原因：主题配置文件中使用了示例占位符

问题二：标签页空白

现象：访问 /tags/ 页面显示空白（0 字节）

原因：缺少 tags.pug 布局文件

解决方案：创建 themes/butterfly/layout/tags.pug

关键点：

  • 使用 Butterfly 主题提供的 cloudTags helper
  • 明确指定 layout: tags 在 front matter 中

问题三：分类页空白

现象：访问 /categories/ 页面显示空白（0 字节）

原因：

  1. 缺少 categories.pug 布局文件
  2. Pug 的 each 语法在特定上下文中不工作

解决方案：创建 themes/butterfly/layout/categories.pug

关键点：

  • 使用 JavaScript forEach 替代 Pug 的 each 语法
  • 先对分类进行排序：.sort('name', 1)
  • 明确指定 layout: categories 在 front matter 中

经验总结

教训

  1. 不要假设主题包含所有布局 - Butterfly 主题需要手动创建 tags/categories 布局
  2. Pug 语法有局限性 - 在复杂上下文中使用 JavaScript 更可靠
  3. 明确指定 layout - 不要依赖 Hexo 的自动推断

最佳实践

  1. 创建布局文件时参考主题文档 - Butterfly 文档提供了布局文件示例
  2. 使用 helper 函数 - cloudTags、url_for 等 helper 简化开发
  3. 测试后再部署 - 本地验证后再推送到 GitHub
