定时任务与自动化

让 Hermes 自动干活 —— 定时任务与自动化

解决的问题：能不能让 Hermes 每天早上自动给我发日报？或者定时帮我做某些事，不用我每次都去叫它？

你有 Hermes，它帮你值班

想象一下这种体验：

每天早上 8:00

你打开手机，看到 Hermes 发来的消息：

“☀️ 早上好！今天的 GitHub 仓库有 3 个新 star，OpenLLM 项目有一个新的 issue。今天天气预报多云，21°C。你的日程中下午 2 点有团队会议。”

你什么都没做——这是 Hermes 自动发给你的。

Hermes 的”自动模式”

Hermes 内置了一个定时任务系统，你可以告诉它：

帮我在每天早上 8 点，把 GitHub 动态和天气情况发到我的 Telegram。

然后它就记住了这个安排，到点自动执行，不需要你管。

能自动做什么？

自动任务 示例
📰 每日简报 每天早上汇总新闻、GitHub 动态、天气
📊 定期报告 每周五下班前生成本周工作总结
🔔 监控提醒 当某个网站更新时通知你
📝 定时提醒 “每周一提醒我交周报”
🗑️ 定期清理 “每周清理一次下载文件夹”

怎么设置？就是聊几句天

设置定时任务简单到你可能不敢相信：

你：以后每天早上 9 点，帮我把 Hacker News 今日热门总结发到我的 Telegram。

Hermes：已收到！我会每天早上 9:00 汇总 Hacker News 热门内容，然后推送到你的 Telegram。

就是这么简单。一句话就设置好了。

几个真实的设置对话

场景 1：日报自动生成

你：每个工作日下班前（下午 6 点），帮我总结今天的完成事项，列个清单。

Hermes：好的，我已经设置了”每日工作日报”自动任务。每个工作日下午 6 点，我会询问你今天完成的事项并整理成清单。

场景 2：周报助手

你：每周五下午 5 点提醒我写周报，并且把这一周每天的日报汇总给我。

Hermes：收到！我设好了每周五下午 5 点的周报提醒，到时我会把你这周的日报汇总给你。

场景 3：关注的项目动态

你：关注一下我 GitHub 上的几个项目，有新的 issue 或者 star 变化就告诉我。

Hermes：我已经记录了你的 GitHub 项目。当有新的动态时我会及时通知你。

所有自动任务都在掌控中

你可以随时查看和管理所有自动任务：

你：我现在有哪些自动任务？

Hermes：目前设置了这些自动任务：

  1. ☀️ 每日简报 — 每天 8:00 → Telegram
  2. 📊 周报提醒 — 每周五 17:00
  3. 🐙 GitHub 动态监控 — 实时通知

要停用或修改哪个？告诉我就行。

你：把第三个停掉吧，最近项目不活跃了。

Hermes：已停用 GitHub 动态监控。需要时可以再启用。

这和你用过的”定时任务”有什么不同

如果你用过 cron 或者自动化工具，你知道传统方式需要：

  1. 写脚本
  2. 配置定时器
  3. 调试
  4. 维护

在 Hermes 这里，就是一句对话的事。

你不需要知道任何技术细节——不需要 cron 表达式，不需要写代码，不需要配置文件。你只需要告诉 Hermes 你想让它做什么、什么时候做。

关于可靠性

你可能会担心：如果我的电脑关机了怎么办？

推荐做法：把 Hermes 部署在一台长期开机的服务器上（VPS 一个月也就几十块钱），这样它就能 7×24 小时工作。

或者如果你用 Telegram 接入，可以把 Hermes 跑在你的电脑上——电脑开着的时候它就会干活。
